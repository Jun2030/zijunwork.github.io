Vue3从零开始集成-1-环境配置

发表于 2020-12-23 分类于 Vue ， Vue3 阅读次数： Valine：
本文字数： 1.5k 阅读时长 ≈ 3 分钟

本篇主要记录-Vue3从零开始集成-1-环境配置。

环境前提：

  • Node.js

  • Yarn/Npm

    此后统一采用yarn。其常用语法对比如下：

    npmyarn
    npm inityarn init
    npm installyarn
    npm install xxx@xxx -gyarn global add xxx@xxx
    npm install xxx@xxx -Syarn add xxx@xxx
    npm install xxx@xxx -Dyarn add -D xxx@xxx
    npm uninstall xxx -Dyarn remove xxx
    npm run xxxyarn run xxx

安装Vue-cli

  1. 安装vue-cli

  2. 借助vue脚手架构建项目

  3. 选择本地磁盘目录：F/simple-vue

  4. 选择[Choose Vue Version]、[Babel]、[TypeScript]、[Router]、[VueX]、[CSS Pre-processors]、[Linter/Formatter]、[使用配置文件]

  5. 选择vue3.x、Less、ESLint+Prettier

  6. UI创建好后，会自动安装依赖（yarn install）,只需运行 yarn serve 即可运行项目

  7. 也可测试生产构建 yarn build，先记录，此后可用于配置（优化）

  8. 至此，提交一个Git版本

升级依赖

根据Webstorm的Node.js和NPM项，可提示最新可用包，升级最新版本并固定版本号，此后所装所有依赖都为固定版本号，因为这样可以保证现有项目运行依赖的一致性，避免依赖升级导致项目错误等问题，如果有依赖需要升级，可手动升级依赖包即可，这样即可测试某依赖包的可用性，也可以保证项目的稳定性。

  1. 升级并固定所有依赖包

  2. 重新安装依赖，运行 yarn install

  3. 此时，运行 yarn dev 可能会报eslint错误，鼠标右键 [src]目录，点击[修复ESLint问题]

  4. 再次测试 yarn serve、yarn build 无问题

  5. 至此，提交一个Git版本
