Node-基本模块

发表于 2020-09-11 分类于 服务端 ， Node ， Node基础 阅读次数： Valine：
本文字数： 3.7k 阅读时长 ≈ 7 分钟

本篇主要记录Node.js-基本模块：url模块、querystring模块、http模块、fs模块、path模块、events模块、global模块。

模块化

  • node中所使用的模块化标准是Commonjs的规范，即 module.exports/require
  • ES6中所使用的模块为新一代规范，即 export/import

Commonjs模块与ES6模块的区别：

  • Commonjs输出是一个值的拷贝，而ES6输出的是值的引用
  • Commonjs是运行时加载，ES6是编译时输出接口

url模块

url 模块用于处理与解析URL。注意，使用时不需要引入 url 模块。

new URL

  • href 获取及设置序列化的URL
  • origin 获取URL的源部分（协议（protocal）+ 主机(host) + 路径(pathname) + 查询(search)）
  • protocal 获取URL的协议
  • username 获取URL的用户名
  • password 获取URL的密码
  • host 获取URL的主机（主机名(hostname) + 端口(port)）
  • hostname 获取URL的主机名
  • port 获取URL的端口
  • pathname 获取URL的路径
  • search 获取URL的查询
  • hash 获取URL的片段
  • searchParams 获取URL查询参数对象
    • get(name) 返回键是name的第一个键值对的值。如果没有对应的键值对，则返回null。
    • tostring() 返回查询参数序列化后的字符串

举例：比如URL http://user:pass@sub.example.com:8080/p/a/t/h?query=string#hash

http模块

  • http.createServer([options][, requestListener]) 创建一个web服务
  • http.request(options[, callback]) 发送HTTP请求
  • response
    • response.writeHead(statusCode[, statusMessage][, headers]) 向请求发送响应头
    • response.end([data[, encoding]][, callback]) 向服务器发出信号，表明已发送所有响应头和主体
    • response.setHeader(name, value) 设置响应头属性
    • response.setTimeout(msecs[, callback]) 设置响应超时时间及事件
    • response.statusCode 当使用隐式的响应头时（没有显式地调用 response.writeHead()），此属性控制在刷新响应头时将发送到客户端的状态码
    • response.statusMessage 当使用隐式的响应头时（没有显式地调用 response.writeHead()），此属性控制在刷新响应头时将发送到客户端的状态消息

fs模块

  • fs.promises.mkdir(path[, options]) 异步地创建目录，然后在成功时解决 Promise，且不带参数
  • fs.promises.opendir(path[, options]) 异步地打开目录
  • fs.promises.open(path[, flags[, mode]]) 异步地打开文件并返回一个 Promise
  • fs.promises.readFile(path[, options]) 异步地读取文件的全部内容
  • fs.createReadStream(path[, options])
  • fs.promises.access(path[, mode]) 测试用户对 path 指定的文件或目录的权限
  • fs.promises.writeFile(file, datap[, options]) 异步地将数据写入到一个文件，如果文件已存在则覆盖该文件
  • fs.promises.appendFile(path, data[, options]) 异步地追加数据到文件，如果文件尚不存在则创建文件
  • fs.promises.stats(path[, options]) 读取文件信息

path模块

  • path.resolve([..paths]) 将路径或路径片段的序列解析为绝对路径
  • path.join([...paths]) 将所有给定的 path 片段连接到一起
  • path.dirname(path) 返回 path 文件的父文件夹
  • path.basename(path) 返回 path 文件名
  • path.extname(path) 返回 path 的扩展名
  • path.parse(path) 返回一个对象，其属性表示 path 的有效元素（dir、root、base、name、ext）

events模块

  • eventEmitter.on() 注册监听器
  • eventEmitter.emit() 触发事件
  • eventEmitter.once() 注册最多只可调用一次的监听器
  • eventEmitter.off() 移除监听器
  • eventEmitter.removeAllListeners() 移除所有监听器

os模块

  • os.hostname() 主机名
  • os.networkInterfaces() 网络接口的详细信息

global模块

  • ——dirname 当前模块的目录名
  • ——filename 当前模块的文件名
  • module module.exports 用于指定一个模块所导出的内容，即可以通过 require() 访问的内容
  • require() 用于引入模块、 JSON、或本地文件。 可以从 node_modules 引入模块
  • URL 新的URL类
  • URLSearchParams 新的URL参数解析类
  • process 进程对象
  • console 打印
