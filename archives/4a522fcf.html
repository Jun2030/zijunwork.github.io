Vite构建

除了上面之前常用的 Vue CLI方式创建项目之外，Vue3.x还可以通过新工具 Vite 来创建项目。 Vite 是一个由原生ES模块驱动的Web开发构建工具，支持模块热更新和按需加载。

项目结构：

CLI项目结构

与 vue2 项目结构类似，

  • public文件夹：可以存放一些不需要经过webpack打包的文件
    • index.html文件：打包编译后的项目入口
  • src文件夹
    • assets目录：静态资源目录
    • components目录：组件目录
    • main.js文件：webpack打包的入口文件

这里的 createApp 方法是Vue3新增的全局API，用来创建一个Vue应用，并挂载到某个DOM节点。参照Vue3和Vue2的代码，从表面上看，createApp 只是改良版的 render 函数，但其实解决了Vue2中一些不太优雅的用法，具体表现为：

  • Vue2中Vue没有“应用”的概念，所谓的应用只是一个 Vue的根实例：new Vue()，当我们需要修改一些全局属性的时候，只能通过Vue对象本身来修改，这样造成的结果就是会污染Vue对象，导致从同一个Vue 创建的实例会共享相同的全局配置
  • Vue3中而是通过 createApp 创建应用，返回的是独立的实例，修改该实例的属性不会影响其他 createApp 创建的实例

TypeScript

Vue3有一个重要的特性就是更好的支持 TypeScript。在使用Vue CLI创建项目的时候，可以选择 TypeScript，这样就能直接创建 ts 项目。如果一开始没有创建 ts 项目，那如何让现有项目支持 ts 呢？有两种方式：

  1. 可以通过命令 vue add typescript 将项目改造为 ts 项目，（前提是电脑上装有Vue CLI）

  2. 手动引入 typescript

    1. npm i typescript -D 安装 typescript 依赖

    2. 项目根目录创建 tsconfig.json 文件：

    3. 将 src 目录下的 main.js 文件名改为 main.ts，以及其他 js 文件改为 ts 后缀，比如：config.js => config.ts

    4. src 根目录新增文件 shim-vue.d.ts

    5. 由于Vue3是使用 TypeScript 编写的，所以只要在 <script> 标签上声明 lang="ts" ，就能直接在 .vue 文件中使用 ts 语法。
