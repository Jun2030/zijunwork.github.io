Koa框架07-CookieSession

发表于 2020-09-21 分类于 服务端 ， Node ， Koa2框架 阅读次数： Valine：
本文字数： 3.5k 阅读时长 ≈ 6 分钟

本篇主要记录Koa框架07-Cookie、Session。

Cookie

Cookie是存储在访问者的计算机中的变量。可以让我们用同一个浏览器访问同一个域名的时候共享数据。

解决的问题是：由于HTTP是无状态的协议，简单的说就是每当浏览了一个页面，然后跳转到同一个网站的另一个页面，服务器无法识别这是同一个浏览器在访问同一个网站，每一次访问，都是无关联的。

使用Cookie

  • ctx.cookies.set(name, value, [options])
    • options 设置的属性
      • maxAge 一个数字，表示从 Date.now() 得到的毫秒数，如果为0，可用于清除cookie。优先级大于 expires。例：1小时：maxAge: 1000 * 60 * 60
      • expires 一个 Date 对象，表示cookie的到期日期（默认情况下会在会话结束时过期）。清除cookie：expires: Date.now() - 1
      • path 一个字符串，表示cookie的路径（默认是“/”）
      • domain 一个字符串，表示cookie的域（无默认值）
      • secure 一个布尔值，表示cookie是否仅通过HTTPS发送（HTTP下默认为 fasle，HTTPS下默认为 true）
      • httpOnly 一个布尔值，表示cookie是否仅通过HTTP/HTTPS发送，且不提供给客户端JavaScript（默认为 true）
      • sameSite 一个布尔值或字符串，表示该cookie是否为相同站点cookie（默认为 false），可以设置为 strict/lax/none/true
      • signed 一个布尔值，表示是否要对cookie进行签名（默认为 false），如果未 true，则还会发送另一个后缀为 .sig 的同名cookie
      • overwrite 一个布尔值，表示是否覆盖以前设置的同名的cookie（默认是 false）
  • ctx.cookie.get(name, [options])

如果没有设置cookie的过期时间，则表示这个cookie生命周期为浏览器会话期间，也就是会话cookie，只要关闭浏览器窗口，cookie就消失了。此外，cookie不会保存在硬盘上而是保存在内存里。

如果设置了过期时间，浏览器就会把cookie保存在硬盘上，直到cookie到期后，消失。

这里还使用到了 encodeURIComponent 和 decodeURIComponent

  • encodeURI：该方法不会对ASCII字母、数字、~!@#$&*()=:/,;?+’字符编码，即会对中文、空格等进行编码
  • encodeURIComponent：该方法不会对ASCII字母、数字、~!*()’字符编码，即会对非字母数字等进行编码，通常对查询字符串编码，也适用于cookie编码

Session

Session是另一种记录客户状态的机制，不同于Cookie的是，Cookie保存在客户端浏览器中，而session保存在服务器上。

原理：

  1. 浏览器访问服务器并发送第一次请求，服务器会创建一个session对象，生成一个（key/vlue）键值对
  2. 响应时，将key(键)通过cookie返回到浏览器客户端
  3. 浏览器非第一次访问，携带key（cookie），服务端找到对应的session（value）

使用Session

先需要安装 koa-session 中间件

  • app.use(session(config, app))
    • config 配置属性
      • key 一个字符串或数字，默认为：koa:sess
      • maxAge 过期时间，以毫秒为单位
      • rolling 一个布尔值，每次浏览器请求时，是否重新刷新session的过期时间，默认为 false
      • renew 一个布尔值，每次浏览器请求时，是否重新刷新session的过期时间，默认为 false

Cookie和Session区别

  1. Cookie数据存在客户端的浏览器上，如果没有设置过期时间，存在内存中，否则存在硬盘上；Session数据则放在服务器上
  2. Cookie不是很安全，别人可以分析存放在本地的cookie并进行欺骗，考虑到安全性，应当使用session
  3. Session会在一定时间内保存在服务器上，当访问比较多，会占用服务器性能，考虑到服务器性能优化，应当适当使用cookie
  4. 当个Cookie保存数据大小不能超过4k，并且浏览器对一个站点的cookie有数量限制，一般限制为20或50个cookie
