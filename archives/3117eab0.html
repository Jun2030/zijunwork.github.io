本篇主要记录-Vue3从零开始集成-8-配置独立化。

背景

在Vue中，如果想要对项目进行统一配置的抽离，一般有3种方法：

  • webpack内js集中化。优势是集中配置，打包后配置不可见（安全），修改方便，但是内嵌与业务代码中，并且修改之后需要前端重新构建。
  • env内进行配置。优势是打包后配置不可见（安全），跟随开发/生产环境修改，独立于业务代码，但是也还是修改之后需要前端/运维重新构建
  • webpack外js集中化。优势是集中配置，修改方便，独立于webpack和业务代码之外，修改之后无需重新构建，但是安全性欠缺。

不同的场景可根据实际选择配置方式。本项目将采用第三种方案。

原理

在app注入脚本之前，预先在全局顶级对象 window 上定义配置，从而可让整个项目中可用

实践

由于有尝试 script 的 type=module 属性，但是测试不可行，原因为：

  • <script type='module' src='xxx'></script>，浏览器加载顺序在 app 挂载之前，但是无法实现 window 挂载
  • <script type='module'> import 'xxx' from 'xxx'</script>, 默认为 defer 属性，浏览器加载顺序在 app 挂载之后。
