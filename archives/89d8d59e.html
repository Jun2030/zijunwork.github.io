Vue3基础-CompositionAPI

发表于 2020-11-03 分类于 Vue ， Vue3 阅读次数： Valine：
本文字数： 6.8k 阅读时长 ≈ 12 分钟

本篇主要记录Vue3基础-CompositionAPI。

介绍

在Vue2中编写Vue组件使用的是选项式API（data、computed、created、mounted、methods…）,当我们页面（组件）变得复杂时，逻辑代码也会增长，我们必须不断地“跳转”相关的选项块，这种碎片化使得理解和维护复杂组件变得困难。随后，采用 Mixin 来解决复用性问题，即 Mixin可以抽取公共逻辑和方法，但 Mixin 提供的数据和函数，无法在组件中直接体现，这导致组件的维护人员需要非常熟悉被引入的 Mixin 对象，不然会出现重命名而导致bug的情况。

基本的周期历程：创建 -> 加载 -> 更新 -> 卸载，Vue3 对比 Vue2：

  • 命名变化：周期函数均以 on 开头命名
  • 废弃创建阶段的周期函数 beforeCreate() 和 created()，使用 setup() 替代
  • 卸载阶段的周期函数重命名：beforeDestory() 调整为 onBeforeUnmount()，destoryed() 调整为 onUnmounted()

跟Vue2中的 beforeCreate、created 一样，在Vue3中 setup 执行时尚未创建组件实例，因此在 setup 选项中没有 this。这意味着，除了 props 之外，setup 中无法访问组件中声明的任何属性（ 本地状态、计算属性、方法等）。

setup

  1. setup() 不是生命周期钩子函数
  2. Vue3的生命周期函数，可以按需导入组件中，且只能在 set() 函数中声明
  3. setup() 显式返回一个对象，这个对象的所有内容都会暴露给组件的其他部分

setup()只是基于 beforeCreate() 运行，但函数内部无法通过 this 获取组件实例，而且 setup() 有着生命周期钩子不具备的参数：props 和 context。

这里的 props 就是组件内部定义的 props，是由父组件传入的参数，假如父组件传入了一个属性名为 msg 的 props，在 setup() 中可以通过 props.msg 直接获取到。而且这个 porps 是响应式的，当传入新的 props 时，props 会被更新，但正是因为 props 是响应式的，所以不能直接对 props 使用ES6的解构，因为它会消除 props 的响应式，Vue3提供了一个 toRefs() 全局方法来解决这个问题。

computed

setup()的本意是用来替代 mixin，因此除了 data、methods、声明周期钩子之外，还有许多组件选项需要解决。

computed 分两种：

  • 组件中的 computed，和Vue2中的用法一致，计算属性可以在组件中直接使用

  • setup() 中的 computed,需要通过全局工具函数 computed 来包装

watch

watch 和 computed 一样，也分两种：

  • 组件中的 watch,和Vue2中的用法一致

  • setup()中的 watch,需要通过全局工具函数 watch来包装
